# تقرير إصلاح سياسات الأمان (SECURITY SQL FIX REPORT)

تم بنجاح مراجعة وهيكلة سياسات الوصول على مستوى الصفوف (Row Level Security) لجميع جداول قاعدة بيانات "شمس التعافي"، وتم إعداد ملف `security_policies_fix.sql` الجاهز للتنفيذ.

### 🛡️ المشاكل التي تم حلها:
1. **خطأ `new row violates row-level security policy` في `profiles`:**
   - **السبب:** لم تكن هناك سياسة (Policy) تسمح للمستخدم أو للأدمن المجهول بإدراج سجل جديد له عند إنشاء الحساب.
   - **الحل:** تمت إضافة `CREATE POLICY "Users can insert own profile" ON profiles FOR INSERT WITH CHECK (auth.uid() = id);`

2. **منع الأهل من رؤية مقيمين غير تابعين لهم:**
   - **السبب:** السياسات القديمة كانت تعتمد على استعلامات بسيطة.
   - **الحل:** ربط الجداول بـ `family_links` عبر `EXISTS` للتحقق من أن العائلة مسموح لها برؤية هذا المقيم تحديداً، وأن الارتباط `is_active = true`.

3. **منع الوصول للتقارير غير المنشورة واليوميات غير المسموحة:**
   - **السبب:** الحاجة لتقييد الـ SELECT بناءً على حالة التقرير.
   - **الحل:** تمت إضافة شرط `visible_to_family = TRUE` لليوميات، وشرط `report_status = 'published'` بالإضافة للارتباط للملفات الخاصة بالتقارير.

4. **تأمين ملفات التخزين (Storage Buckets):**
   - **السبب:** كانت ملفات الـ Media والـ Reports عرضة للوصول.
   - **الحل:** تم تقييد الوصول لكل Bucket بحسب نوعه (Public أو Private)، حيث لا يمكن للأهل رؤية الملفات الخاصة إلا للمقيمين المرتبطين بهم والمسموح لهم بذلك (`can_view_reports = true`).

5. **تحديد صلاحيات الحذف والتعديل للموظفين (Staff):**
   - **السبب:** عدم توحيد دالة فحص الأدوار.
   - **الحل:** استخدام الدالة المساعدة `auth_user_has_role` لضمان أن `super_admin, admin, staff, doctor` فقط هم من يمتلكون حق `FOR ALL` على جداول المرضى.

---

### 🗃️ الجداول التي تم تأمينها بنجاح بالـ RLS:
- `profiles`
- `residents`
- `family_links`
- `resident_updates`
- `weekly_reports`
- `sessions_attendance`
- `news`, `gallery`, `messages`, `notifications`
- `audit_logs` (فقط للمشاهدة من قبل الأدمن، الإدخال يتم عبر Triggers)
- `settings`, `branches`, `rooms`
- `storage.objects` (ملفات التخزين)

---

### ⚙️ طريقة التنفيذ:
1. اذهب إلى لوحة تحكم **Supabase**.
2. من القائمة الجانبية، اختر **SQL Editor**.
3. انقر على **New query**.
4. افتح الملف المُرفق `security_policies_fix.sql` وانسخ جميع محتوياته.
5. الصقها في المحرر واضغط على زر **RUN** المضاء باللون الأخضر.
*(هذا السكربت آمن 100٪؛ حيث يستخدم `DROP POLICY IF EXISTS` لإزالة السياسات القديمة فقط دون المساس ببياناتك وجداولك).*

---

### 🧪 اختبارات سريعة بعد التنفيذ:
- **اختبار الإضافة:** جرب إنشاء حساب أهل جديد أو مقيم من הـ Dashboard (بعد تسجيل دخولك كأدمن). لن يظهر خطأ `violates RLS`.
- **اختبار الخصوصية:** جرب الدخول بحساب "أسرة"، ستلاحظ أن صفحة "المقيم التابع لي" لن تعرض سوى المقيم المرتبط فقط.
- **اختبار التقارير:** التقرير المسجل كـ `draft` لن يظهر لحسابات الأهل حتى يتم تحويله إلى `published`.
